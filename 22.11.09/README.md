### 트랜잭션
* 일괄로 통채로 관리(예를 들면 패키지 여행 상품)

* 트랜잭션이 많아질수록 잠금이 많아짐.

#### 교착상태
여러 개의 트랜잭션(Transaction)들이 실행을 하지 못하고 서로 무한정 기다리는 상태
* 예방 방법으로는 크게 두 가지 : 

	1.각 트랜잭션이 실행되기 전에 필요한 데이터를 모두 로킹(Locking) 해주는 것
	
	2.자원을 할당할 때 시간 스탬프(Time Stamp)를 사용하여 교착상태가 일어나지 않도록 회피하는 방법
